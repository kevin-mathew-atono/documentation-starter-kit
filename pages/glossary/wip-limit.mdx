# WIP Limit

A maximum number of work items permitted to be simultaneously active in a single workflow step or across the entire In Progress category. WIP limits are a core Kanban practice that force teams to finish work before starting new items, reducing context switching and identifying process bottlenecks. In Atono, workflow steps can be configured with WIP limits visible on the In Progress page.
