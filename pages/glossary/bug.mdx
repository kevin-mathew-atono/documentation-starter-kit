# Bug

An error, flaw, or unintended behavior in software that requires investigation and resolution. Each bug includes a title, description (summary, reproduction steps, expected and actual behavior), priority, severity, affected environments, and an optional assignee. Bugs move through team workflows and support threaded comments, subtasks, attachments, and version history. Triage happens before a bug is assigned to a team for active work.
