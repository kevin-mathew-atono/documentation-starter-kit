# Bug

An error, flaw, or unintended behavior in software requiring investigation and resolution. Each bug includes a title, structured description (summary, reproduction steps, expected behavior, actual behavior), risk rating calculated from probability × impact (1–25 scale), affected environments, and an optional assignee. Bugs move through team workflows, participate in Bug Triage before team assignment, and support cycle time tracking, duplication, version history, threaded comments, linked items, subtasks, and attachments.
