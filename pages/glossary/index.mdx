# Glossary

A complete reference for every concept and feature in Atono — 122 entries.

## A

- [Acceptance Criteria](/glossary/acceptance-criteria) — Testable conditions that define when a story is complete, written as a hierarchical list of requirements.
- [Active Sprint](/glossary/active-sprint) — The current sprint a Scrum team is executing, visible on the team's In Progress page.
- [Activities](/glossary/activities) — A chronological, filterable log of all changes to a story, bug, epic, or feature flag including title changes, workflow transitions, assignments, comments, attachments, subtasks, and pull requests.
- [Additional Content](/glossary/additional-content) — A freeform rich-text section in a story below the acceptance criteria, used for background information, technical notes, or context that doesn't fit in the user story or ACs.
- [Administrator](/glossary/administrator) — A workspace role with full administrative privileges including managing workspace settings, environments, users, integrations, and SSO configuration, but without access to billing.
- [Affected Environment](/glossary/affected-environment) — One or more environments where a bug has been observed, selected from the workspace's configured environments such as development, staging, or production.
- [AI Context](/glossary/ai-context) — A structured section on a story or bug where AI tools store and retrieve context across work sessions, divided into Design Decisions, Technical Investigation, and Technical Changes.
- [API Key](/glossary/api-key) — A user-specific credential for authenticating with the Atono public API, created and managed from the user's profile.
- [Ask Capy](/glossary/ask-capy) — An AI assistant that answers questions about your product by searching completed stories and bugs in your workspace.
- [Assignee](/glossary/assignee) — The user responsible for completing a story or bug.
- [Atono API](/glossary/atono-api) — The public REST API for programmatic access to Atono data and operations, authenticated using API Keys.
- [Atono Chrome Extension](/glossary/atono-chrome-extension) — A Chrome browser extension that enables bug reporting with automatic context capture, feature flag flipping, and click mapping for usage tracking directly from your application.
- [Atono MCP Server](/glossary/atono-mcp-server) — A local Docker-based server implementing the Model Context Protocol that connects AI development tools like Cursor, Claude, or VS Code to an Atono workspace.
- [Attachment](/glossary/attachment) — A file added to a story or bug, displayed as thumbnails for images/videos or as file info for other types.

## B

- [Backlog (Page)](/glossary/backlog-page) — A team page for planning and prioritizing upcoming work before it moves into execution.
- [Backlog Management](/glossary/backlog-management) — A team setting that, when enabled, assigns backlog owners responsibility for managing upcoming stories including adding, creating, reordering in To do, deleting, and moving in/out of Won't do.
- [Backlog Owner](/glossary/backlog-owner) — A team role responsible for managing a team's planned work when Backlog Management is enabled.
- [Bug](/glossary/bug) — An error, flaw, or unintended behavior in software requiring resolution.
- [Bug Triage](/glossary/bug-triage) — A workspace-wide process and page for evaluating and prioritizing reported bugs before assigning them to teams.
- [Burndown Chart](/glossary/burndown-chart) — A sprint-level chart for Scrum teams showing story points remaining over time compared to an ideal guideline.

## C

- [Capy](/glossary/capy) — Atono's capybara mascot representing calm, collaborative energy and team spirit.
- [Comment](/glossary/comment) — Feedback or discussion added to a story or bug, either generally or on specific highlighted text or acceptance criteria.
- [Completed Date](/glossary/completed-date) — The date an item was last moved into a workflow step in the Done category.
- [Concept](/glossary/concept) — An individual entry in the Product Knowledge glossary, typically including a name, plain-English description, synonyms/aliases, and related concepts.
- [Cycle Time](/glossary/cycle-time) — The total time a story or bug spends in workflow steps categorized as In Progress for a specific team.
- [Cycle Time Report](/glossary/cycle-time-report) — A team-specific report visualizing average cycle times for stories and bugs over a selected timeframe (1-60 months).

## D

- [Design Decisions](/glossary/design-decisions) — An AI Context section capturing product and scope decisions made during story authoring, including tradeoffs, AC framing, scope boundaries, and deferred items.
- [Docker](/glossary/docker) — Container platform required to run the Atono MCP Server locally.
- [Done (Workflow Category)](/glossary/done-workflow-category) — A workflow category containing steps representing completed work.
- [Draft](/glossary/draft) — An unsaved version of a story or bug being created, displayed in a partial overlay that allows navigation while editing.
- [Duplicate](/glossary/duplicate) — A copy of an existing story or bug created with the same title (prefixed 'Copy of'), description, risk rating (bugs), and size/product theme (stories).

## E

- [Engagement (Page)](/glossary/engagement-page) — A workspace page showing a Top Features graph that ranks up to 10 features by usage across the product.
- [Environment](/glossary/environment) — A named stage in the software development lifecycle such as dev, test, stage, or prod, used to manage feature flags and track bugs.
- [Environment Key](/glossary/environment-key) — A unique identifier for an environment, used to connect application code to Atono for feature flag evaluation and usage tracking.
- [Epic](/glossary/epic) — A planning item that groups up to 100 related stories under a shared goal.
- [Estimated Completion Date](/glossary/estimated-completion-date) — A forecast for when items in a Kanban team's To do category will be completed, based on the team's average cycle time and the current backlog queue order.
- [Everything (Page)](/glossary/everything-page) — A centralized, filterable view of all stories, bugs, and epics in the workspace that the user has permission to view.
- [External ID](/glossary/external-id) — An identifier from another system (e.

## F

- [Feature Engagement](/glossary/feature-engagement) — A story-level usage graph showing how often a feature is used, filterable by environment, customer, or location.
- [Feature Flag](/glossary/feature-flag) — A named toggle controlling feature visibility and behavior across environments without code deployment.
- [Filter](/glossary/filter) — Criteria applied to narrow visible items in backlogs, the Everything page, timelines, or reports.
- [Flag Configuration](/glossary/flag-configuration) — The settings controlling a feature flag's behavior in each environment.
- [Flag Owner](/glossary/flag-owner) — A permission level granting full control over a feature flag including configuration, naming, permissions management, removal from stories, and deletion.
- [Flag Permissions](/glossary/flag-permissions) — Access controls for feature flags, separate from workspace and team roles.
- [Flag Status](/glossary/flag-status) — A summary of a feature flag's current configuration and evaluation state across all stories and environments.
- [Flag User](/glossary/flag-user) — A permission level allowing configuration and toggling of feature flags on/off and defining slices, but not editing flag name/description, managing permissions, removing from stories, or deleting.
- [Follow](/glossary/follow) — An action marking a story or bug (via star icon) so it appears in filtered views and dashboard widgets.

## G

- [Glossary](/glossary/glossary) — A structured, exportable collection of up to 200 key product concepts built from documentation by Product Knowledge.
- [Glossary Contributor](/glossary/glossary-contributor) — A permission level allowing users to add, edit, delete, and restore concepts in the Product Knowledge glossary.
- [Glossary Owner](/glossary/glossary-owner) — A permission level with Glossary Contributor abilities plus glossary-level permissions management and deletion rights.

## H

- [Home (Page)](/glossary/home-page) — A user's landing page in Atono showing widgets for assigned work, followed items, and any saved/shared views added as widgets.

## I

- [Identity Provider](/glossary/identity-provider) — An external system managing user identities and authentication for an organization, used with SSO.
- [Impact](/glossary/impact) — The severity of a bug's effect on users, rated 1-5 (Negligible to Severe) during Bug Triage.
- [Import](/glossary/import) — The process of bringing existing work from Jira or Linear into Atono, including issues, teams, projects, and optionally users.
- [In Progress (Page)](/glossary/in-progress-page) — A team page showing active work moving through workflow steps.
- [In Progress (Workflow Category)](/glossary/in-progress-workflow-category) — A workflow category containing steps representing active work.
- [Item ID](/glossary/item-id) — A unique identifier assigned to each story, bug, or epic in the format TYPE-NUMBER (e.

## K

- [Kanban](/glossary/kanban) — A team workflow type focused on continuous flow rather than time-boxed iterations.

## L

- [Last Evaluated](/glossary/last-evaluated) — The most recent timestamp when a feature flag was evaluated (checked for ON/OFF state) by the SDK across any environment.
- [Last Updated](/glossary/last-updated) — The date an item's properties were last modified, including title, backlog position, assignee, description/ACs, feature flags, team, risk rating, size, workflow step, timebox, or product theme.
- [Linked Items](/glossary/linked-items) — Stories, bugs, or epics explicitly connected to each other to show relationships like dependencies or related work.

## M

- [Mapped Action](/glossary/mapped-action) — A recorded association between a user interaction in an application and a story in Atono, created through click mapping in the Atono Chrome extension.
- [MCP](/glossary/mcp) — Model Context Protocol, an open protocol allowing AI tools to securely connect to external systems and use their data and actions in context.
- [Meta Tag](/glossary/meta-tag) — An HTML metadata snippet in a webpage's &lt;head&gt; section providing information to tools.

## N

- [Notification](/glossary/notification) — An email sent to users when specific events occur, such as being assigned work or @mentioned in a comment.

## O

- [Outlier](/glossary/outlier) — A story or bug with unusually long cycle time, marked to exclude it from average cycle time calculations, cycle time reports, staleness indicators, and estimated completion dates.

## P

- [Persona](/glossary/persona) — A named user type in a user story's 'As a [persona].
- [Probability](/glossary/probability) — The likelihood that a user will encounter a bug, rated 1-5 (Impossible to Certain) during Bug Triage.
- [Product Knowledge](/glossary/product-knowledge) — A feature that analyzes help documentation and builds a structured glossary of up to 200 key product concepts including name, description, synonyms, and relationships.
- [Product Manager](/glossary/product-manager) — A workspace role controlling who can manage product themes.
- [Product Theme](/glossary/product-theme) — A named grouping for related stories aligning work with strategic goals and tracking long-term progress.
- [Projected Completion](/glossary/projected-completion) — An estimated date when a story or bug will be completed, calculated differently for Kanban and Scrum teams.
- [Pull Request](/glossary/pull-request) — A proposed code change in Git-based version control systems, automatically linked to stories or bugs when mentioned by ID in the PR title, description, branch name, or optionally comments and commit messages.

## R

- [Release](/glossary/release) — A key delivery milestone represented by a date, created as release markers on timelines.
- [Reporter](/glossary/reporter) — The user recorded as reporting a bug, which may differ from the user who created the bug in Atono.
- [Risk Rating](/glossary/risk-rating) — A numerical assessment of a bug's overall risk, calculated by multiplying probability (1-5) by impact (1-5) to produce a value from 1-25, where 25 is highest risk.
- [Roles and Permissions](/glossary/roles-and-permissions) — Atono's multi-level access control system including workspace roles (Workspace Owner, Administrator, Product Manager, Standard User), team roles (Team Member, Team Administrator, Backlog Owner), and separate feature flag and glossary permissions.

## S

- [SAML](/glossary/saml) — Security Assertion Markup Language, a standard for exchanging authentication information between an identity provider and service provider.
- [Saved View](/glossary/saved-view) — A named, personal collection of filters, sorting, and column customizations on the Everything page that can be returned to without reapplying settings.
- [Scrum](/glossary/scrum) — A team workflow type focused on time-boxed iterations called sprints.
- [SDK](/glossary/sdk) — Software Development Kit, a set of tools and libraries from Atono for integrating feature flags and usage tracking into applications.
- [Search](/glossary/search) — A fuzzy, keyword-based search across story, bug, and epic titles, IDs, descriptions, user stories, acceptance criteria, and active comments.
- [Shared View](/glossary/shared-view) — A saved view that has been shared with specific users or teams, appearing in their shared views list.
- [Slack Integration](/glossary/slack-integration) — A workspace integration connecting Atono to a Slack workspace, enabling Slack channels for teams and stories, posting updates for team changes and feature flag modifications, and user-level Slack account connection.
- [Slice](/glossary/slice) — A defined subset within an environment for a feature flag, based on customer and/or location attributes with specific operators (is, is not, contains, is one of, is in, is not in).
- [Snooze](/glossary/snooze) — An action that temporarily hides the staleness indicator for a specific item in a specific workflow step for a chosen duration (1 day, 1 week, 1 month, or until changed).
- [Split](/glossary/split) — An action dividing a story into two stories by moving selected acceptance criteria into a new story.
- [Sprint](/glossary/sprint) — A time-boxed iteration for Scrum teams, typically 1-4 weeks, with defined start/end dates and committed scope.
- [Sprint Calendar](/glossary/sprint-calendar) — A Scrum-only team page providing a visual schedule of past, active, and future sprints.
- [SSO](/glossary/sso) — Single sign-on, an authentication method allowing users to sign in to Atono using their organization's identity provider instead of email-based login links.
- [SSO Mode](/glossary/sso-mode) — A workspace setting controlling how SSO is applied: On (all users use SSO), Off (all users use email login), or Testing (only Workspace Owners and Administrators use SSO).
- [Stakeholder](/glossary/stakeholder) — A user with a critical team role (Team Administrator or Backlog Owner) who is not a team member.
- [Staleness Indicator](/glossary/staleness-indicator) — A red hourglass icon appearing on stories or bugs in In Progress steps when time spent exceeds 3 standard deviations of the team's average for similar items in that step.
- [Standard User](/glossary/standard-user) — The default workspace role assigned to all users, providing typical functionality including creating teams, managing public team backlogs (without backlog owners), creating/managing bugs and stories on public teams, and using most Atono features.
- [Story](/glossary/story) — A unit of user-focused work including title, one or more user stories, hierarchical acceptance criteria, and optional additional content.
- [Story Points](/glossary/story-points) — A numerical measure derived from story size (XXS=1, XS=2, S=3, M=5, L=8, XL=13, XXL=21) using Fibonacci numbers.
- [Story Refinement](/glossary/story-refinement) — A workspace-wide backlog for capturing ideas and preparing stories before assigning them to teams.
- [Story Size](/glossary/story-size) — A relative measure of effort to complete a story: Unsized, XXS, XS, S, M, L, XL, XXL.
- [Subtask](/glossary/subtask) — An implementation step within a story or bug with title, optional description (up to 1000 chars with rich text), assignee, and status (To do, In progress, Done, Won't do).

## T

- [Team](/glossary/team) — A group of users organized to plan and deliver work together in Atono, using either Scrum or Kanban workflow.
- [Team Administrator](/glossary/team-administrator) — A team role with permissions to edit team details, delete team, modify access type and methodology, manage team admins and backlog owners, customize workflow, and connect Slack channels.
- [Team Member](/glossary/team-member) — A team role granted automatically when a user joins a public or private team.
- [Team Role](/glossary/team-role) — Permissions granted within a specific team (Team Member, Team Administrator, Backlog Owner) that do not affect workspace-level access.
- [Technical Changes](/glossary/technical-changes) — An AI Context section recording current implementation changes made by AI tools for a story or bug.
- [Technical Investigation](/glossary/technical-investigation) — An AI Context section capturing discoveries, research findings, and constraints gathered during AI-assisted implementation.
- [Timebox](/glossary/timebox) — A named, date-bounded container grouping stories and bugs into a defined delivery window.
- [Timeline](/glossary/timeline) — A visual schedule of timeboxes created and owned by a user, showing how work is organized over time.
- [To do (Workflow Category)](/glossary/to-do-workflow-category) — A workflow category containing one step representing work ready to start but not yet active.
- [Total Story Points](/glossary/total-story-points) — The combined Fibonacci-based size of all visible stories in a column, sprint, or timebox, including an estimated size for unsized stories based on team average.

## U

- [Usage Tracking](/glossary/usage-tracking) — Recording user interactions with features to power Feature Engagement graphs.
- [User Profile](/glossary/user-profile) — A page showing user details including email, name, workspace role, and team memberships.
- [User Story](/glossary/user-story) — A brief description of a feature from the user's perspective in the format 'As a [persona], I want to [action] so that [benefit].

## V

- [Velocity](/glossary/velocity) — The average number of story points a Scrum team completes per sprint, calculated from completed sprints within the team's configured averaging window.
- [Version History](/glossary/version-history) — A record of all changes to a story or bug's content (title, user stories, acceptance criteria, description) showing additions (green) and deletions (red strikethrough).

## W

- [Widget](/glossary/widget) — A customizable panel on the Home page displaying items from a saved or shared view.
- [Won't do (Workflow Category)](/glossary/wont-do-workflow-category) — A workflow category containing one step for work that will not be completed.
- [Workflow](/glossary/workflow) — A customizable sequence of 4-25 steps that stories and bugs move through from ready-to-start to completion.
- [Workflow Category](/glossary/workflow-category) — One of four organizational groupings for workflow steps: To do (1 step), In Progress (1+ steps, contributes to cycle time), Done (1+ steps), Won't do (1 step).
- [Workflow Step](/glossary/workflow-step) — A named stage in a workflow with assigned color and category (To do, In progress, Done, Won't do).
- [Workspace](/glossary/workspace) — The top-level container for all teams, stories, bugs, users, and other elements in Atono for a company.
- [Workspace Owner](/glossary/workspace-owner) — A workspace role with highest-level permissions including all Administrator capabilities plus managing plans, billing, adding/removing other Workspace Owners, and deleting the workspace.
- [Workspace Role](/glossary/workspace-role) — A primary permission level defining system-wide access: Workspace Owner (highest), Administrator (full admin without billing), Product Manager (theme/glossary management), or Standard User (default).
