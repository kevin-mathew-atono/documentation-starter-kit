# Definition of Done

A shared team agreement listing all criteria that must be satisfied before a story or bug is considered complete. Common criteria include code reviewed, tests written, documentation updated, and acceptance criteria verified. The definition of done ensures consistent quality across the team and reduces ambiguity about when work can be marked as finished.
