# Definition of Done

A team-wide agreement specifying all conditions a story or bug must satisfy before it can be marked complete. Criteria typically include code review, automated tests passing, acceptance criteria verified, and documentation updated. The definition of done is applied consistently across all work items and is distinct from acceptance criteria, which are story-specific. Teams can view and edit their definition of done in team settings.
