# Deployment Pipeline

An automated sequence of steps—such as build, test, and release—that code changes move through from commit to production. Deployment pipelines enforce quality gates and reduce the risk of releasing broken code. Feature flags in Atono complement deployment pipelines by enabling code to be deployed while keeping features disabled until they are ready for users.

## Also Known As

- CI/CD Pipeline
- Release Pipeline
