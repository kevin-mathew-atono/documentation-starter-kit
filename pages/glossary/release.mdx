# Release

A key delivery milestone represented by a date, created as release markers on timelines. Release names must be unique within a timeline. Releases can be surfaced in team backlogs: Scrum backlogs show releases as horizontal rows between sprints; Kanban backlogs let you select one release to highlight items projected to complete within that window.
