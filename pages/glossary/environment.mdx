# Environment

A named stage in the software development lifecycle such as dev, test, stage, or prod, used to manage feature flags and track bugs. Each environment has a unique key for SDK integration. Workspaces include four default environments that can be renamed, reordered, or deleted. New environments can be created by Workspace Owners and Administrators.
