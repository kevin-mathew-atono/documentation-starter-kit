# Velocity

The average number of story points a Scrum team completes per sprint, calculated using only the team's most recent completed sprints within the configured velocity averaging window. Teams with fewer completed sprints than the window size use all available sprints. Velocity is used to project burndown chart completion, estimate future sprint capacity, and calculate estimated completion dates for the Kanban backlog queue.
