# Story

The core unit of user-focused work in Atono, capturing what needs to be built and why. Each story includes a title, one or more user stories in 'As a / I want to / So that' format, hierarchical acceptance criteria, an optional additional context section, story size, product theme, assignee, epic, timebox, and linked feature flags. Stories move through a team's workflow steps, track cycle time, support version history, comments, attachments, subtasks, and linked items, and can be duplicated or split into multiple stories.
