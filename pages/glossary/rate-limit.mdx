# Rate Limit

A cap on the number of API requests a single user or API key can make to Atono within a defined time window. Rate limiting protects platform stability and ensures fair resource distribution across all workspace users. Requests exceeding the limit receive an HTTP 429 Too Many Requests response and should be retried with exponential backoff.

## Also Known As

- API Rate Limit
- Request Throttle
