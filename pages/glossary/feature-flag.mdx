# Feature Flag

A named toggle controlling feature visibility and behavior across environments without code deployment. Flags are created or applied to stories, configured per environment with optional slicing by customer or location, and evaluated by the SDK. Include unique name (4-30 chars, lowercase/underscores), description, ownership permissions, and configuration history. Maximum one flag owner required.
